🎯 Visión General

¿Qué hace esta aplicación?

Es una API REST que permite a un cliente de BTG Pactual manejar sus fondos de inversión sin llamar a un asesor.

💰
Suscribirse a Fondos
El cliente elige un fondo del catálogo y decide cuánto dinero invertir. El sistema valida montos mínimos y saldo disponible.
🔙
Cancelar Fondos
El cliente puede retirar su dinero de un fondo. El sistema calcula cuánto le corresponde y lo devuelve a su saldo.
📋
Ver Historial
El cliente consulta todas sus transacciones: qué fondos suscribió, cuándo y por cuánto dinero.
💡 Analogía para entender
Imagina que esta API es como un cajero automático para fondos de inversión. Te logueas (metes tu tarjeta), ves los fondos disponibles (lees el menú), metes plata en un fondo (depósito) o la sacas (retiro), y siempre puedes ver tu recibo (historial).
🏗️ Arquitectura

Arquitectura por Capas

El proyecto organiza el código en capas. Cada capa tiene una responsabilidad única. Una petición viaja de arriba hacia abajo.

🌐
Controller (Puerta de entrada)
Recibe las peticiones HTTP del cliente (JSON) y las reenvía al Service. No tiene lógica de negocio.
⬇️
⚙️
Service (Cerebro)
Aquí vive toda la lógica del negocio: validar montos, descontar saldos, crear transacciones, enviar notificaciones.
⬇️
🗄️
Repository (Acceso a datos)
Se comunica con MongoDB para guardar y consultar datos. Spring genera las consultas automáticamente.
⬇️
📦
Domain (Entidades)
Define la estructura de los datos (User, Fund, Transaction) que se guardan en la base de datos MongoDB.
📦 Capa Domain

Entidades (Los "objetos" de la BD)

Cada entidad es una clase Java que representa una colección (tabla) en MongoDB. Usan Lombok para no escribir getters/setters manualmente.

👤
User.java
Colección: users

Campos:
• id — Identificador único
• username — Nombre de usuario
• password — Contraseña (encriptada)
• roles — Permisos (ej: "USER")
• balance — Saldo disponible ($)
• email — Correo electrónico
• phone — Teléfono
• notificationPreference — EMAIL o SMS
💼
Fund.java
Colección: funds

Campos:
• id — Identificador único
• nombre — Nombre del fondo
• montoMinimo — Monto mínimo para invertir
• categoria — FPV o FIC

📌 Se crean 5 fondos automáticamente al arrancar.
📄
Transaction.java
Colección: transactions

Campos:
• id — Identificador único
• fundId — A qué fondo se refiere
• userId — Quién hizo la operación
• type — SUBSCRIBE o UNSUBSCRIBE
• amount — Monto de la operación
• timestamp — Fecha y hora
📦 Capa Domain

Enums (Constantes con nombre)

Los enums son listas fijas de valores posibles. Evitan que alguien escriba "suscribe" en vez de "SUBSCRIBE".

🔄
TransactionType.java
Define los dos tipos de operación posibles:
public enum TransactionType {
    SUBSCRIBE,    // El usuario mete plata en un fondo
    UNSUBSCRIBE  // El usuario saca plata de un fondo
}
🔔
NotificationPreference.java
Define cómo quiere ser notificado el usuario:
public enum NotificationPreference {
    EMAIL,  // Notificar por correo
    SMS    // Notificar por mensaje de texto
}
💡 ¿Para qué sirven los enums?
Piensa en un semáforo: solo puede ser ROJO, AMARILLO o VERDE. No puede ser "azul". Los enums son exactamente eso: valores predefinidos que no pueden ser otra cosa.

🗄️ Capa Repository

Repositorios (Acceso a MongoDB)

Los repositorios son interfaces que extienden MongoRepository. Spring genera automáticamente el código para guardar, buscar y eliminar documentos. ¡No escribimos SQL ni queries!

👤
UserRepository.java
Métodos heredados gratis:
• save(), findById(), findAll(), delete()...

Método custom:
• findByUsername(String)
→ Spring lee el nombre del método y genera la query solo.
💼
FundRepository.java
Métodos heredados gratis:
• save(), findById(), findAll(), count()...

No necesita métodos custom porque los fondos se buscan por ID o se listan todos.
📄
TransactionRepository.java
Métodos custom:
• findByUserId(String)
→ Busca transacciones de un usuario

• findByUserIdAndFundId(String, String)
→ Busca transacciones de un usuario en un fondo específico
💡 ¿Cómo funciona la magia?
Si nombras el método findByUsername, Spring entiende: "busca documentos donde el campo username sea igual al parámetro". Si pones findByUserIdAndFundId, busca por dos campos. ¡Solo con el nombre!

public Transaction subscribe(String username, SubscribeRequest request) {
  User user = userRepository.findByUsername(username)
    .orElseThrow(() -> new ResourceNotFoundException("User not found"));

  Fund fund = fundRepository.findById(request.getFundId())
    .orElseThrow(() -> new ResourceNotFoundException("Fund not found"));

  if (request.getAmount().compareTo(fund.getMontoMinimo()) < 0)  // ¿Monto menor al mínimo?
    throw new InsufficientBalanceException("El monto debe ser al menos...");

  if (user.getBalance().compareTo(request.getAmount()) < 0)   // ¿No tiene suficiente plata?
    throw new InsufficientBalanceException("No tiene saldo disponible...");

  user.setBalance(user.getBalance().subtract(request.getAmount()));  // Resta el monto
  userRepository.save(user);          // Guarda el usuario actualizado en MongoDB

  // Crea y guarda la transacción + envía notificación
}

List<Transaction> transactions = transactionRepository.findByUserIdAndFundId(userId, fundId);

// Calcula cuánto tiene invertido realmente:
// SUBSCRIBE $100,000 → amountToReturn = +100,000
// SUBSCRIBE $50,000 → amountToReturn = +150,000
// UNSUBSCRIBE $80,000 → amountToReturn = +70,000 (ya retiró 80K antes)

BigDecimal amountToReturn = BigDecimal.ZERO;
for (Transaction t : transactions) {
  if (t.getType() == SUBSCRIBE)    amountToReturn = amountToReturn.add(t.getAmount());
  if (t.getType() == UNSUBSCRIBE) amountToReturn = amountToReturn.subtract(t.getAmount());
}

if (amountToReturn <= 0) throw new ActiveSubscriptionNotFoundException(...); // No tiene nada invertido

user.setBalance(user.getBalance().add(amountToReturn)); // Devuelve la plata

Métodos: getAllFunds() y getHistory()

Dos métodos simples de consulta que no modifican datos.

public List<Transaction> getHistory(String username) {
  User user = userRepository
    .findByUsername(username)
    .orElseThrow(...);
  return transactionRepository
    .findByUserId(user.getId());
}
⚙️ AuthService.java

Servicio de Autenticación

Maneja el login del usuario y la creación del usuario demo para pruebas.

🔐
authenticate(AuthRequest)
¿Qué hace paso a paso?

1️⃣ Recibe username y password
2️⃣ AuthenticationManager valida las credenciales contra MongoDB (la contraseña se compara con BCrypt)
3️⃣ Si es válido, carga los datos del usuario
4️⃣ Genera un token JWT firmado
5️⃣ Devuelve el token al cliente

⚠️ Si la contraseña es incorrecta, Spring lanza una excepción automáticamente.
👤
createDemoUser()
¿Qué hace?

Crea un usuario de prueba si no existe:

• Username: demo
• Password: password (encriptada con BCrypt)
• Balance: $500,000
• Email: demo@test.com
• Notificación: EMAIL

Se ejecuta al arrancar la app.
⚙️ NotificationService.java

Servicio de Notificaciones

Envía notificaciones al usuario según su preferencia. Actualmente es una simulación (imprime en consola), pero la estructura está lista para conectar un servicio real.

📨
sendNotification()
Método público. Revisa la preferencia del usuario y llama a sendEmail() o sendSms().
📧
sendEmail() (privado)
Simula el envío de un email. En producción se conectaría a AWS SES o similar.
📱
sendSms() (privado)
Simula el envío de un SMS. En producción se conectaría a AWS SNS o Twilio.
public void sendNotification(User user, String message) {
  if (user.getNotificationPreference() == EMAIL) {
    sendEmail(user.getEmail(), message);
  } else if (user.getNotificationPreference() == SMS) {
    sendSms(user.getPhone(), message);
  }
}
🌐 Capa Controller

Controladores (Los endpoints)

Son las "puertas" de la API. Reciben las peticiones HTTP y llaman al servicio correspondiente.

Método HTTP Ruta Controlador ¿Qué hace? ¿Necesita JWT?
POST /api/auth/login AuthController Inicia sesión y devuelve un token JWT ❌ No
GET /api/funds FundController Lista todos los fondos disponibles ✅ Sí
POST /api/funds/subscribe FundController Suscribe al usuario a un fondo ✅ Sí
POST /api/funds/unsubscribe FundController Cancela la suscripción a un fondo ✅ Sí
GET /api/funds/history FundController Muestra el historial de transacciones ✅ Sí
💡 ¿Qué es Authentication authentication?
Cuando un endpoint tiene el parámetro Authentication, Spring automáticamente le pasa la información del usuario autenticado (extraída del token JWT). Con authentication.getName() obtenemos el username sin pedirlo en el body.

🔒 Capa Security

Las 5 clases de seguridad

Cada clase tiene un rol específico en el flujo de autenticación y autorización.

🛡️
SecurityConfig.java
¿Qué configura?
• Desactiva CSRF (no es necesario en APIs REST)
• /api/auth/** es público (sin login)
• Todo lo demás requiere autenticación
• Sesiones STATELESS (sin cookies)
• Registra el filtro JWT antes del de Spring
🔍
JwtAuthenticationFilter.java
¿Qué hace?
Se ejecuta en cada petición.
1. Lee el header Authorization: Bearer xxx
2. Extrae el username del token
3. Verifica que el token sea válido y no haya expirado
4. Si es válido, autentifica al usuario en el contexto de Spring
🔑
JwtService.java
Métodos:
• generateToken() — Crea un JWT
• extractUsername() — Lee el username del token
• isTokenValid() — ¿Token OK y no expirado?
• getSignInKey() — Obtiene la llave secreta para firmar
👥
UserDetailsServiceImpl.java
¿Qué hace?
Implementa UserDetailsService de Spring.
Busca al usuario en MongoDB y lo convierte al formato que Spring Security entiende (con username, password y roles).
⚙️
AppConfig.java
Configura los "beans" necesarios para la seguridad:
• AuthenticationProvider — Usa DaoAuthenticationProvider con nuestro UserDetailsService y BCrypt
• AuthenticationManager — Lo que Spring usa para validar credenciales
• PasswordEncoder — BCryptPasswordEncoder para encriptar contraseñas
⚠️ Capa Exception

Manejo Global de Errores

En vez de que el servidor devuelva un error feo de Java, capturamos los errores y devolvemos mensajes amigables en JSON.

ResourceNotFoundException
Se lanza cuando no se encuentra un usuario o fondo.
→ Devuelve HTTP 404
InsufficientBalanceException
Se lanza cuando el monto es menor al mínimo o no hay saldo suficiente.
→ Devuelve HTTP 400
ActiveSubscriptionNotFoundException
Se lanza cuando intentas cancelar un fondo al que no estás suscrito.
→ Devuelve HTTP 400
GlobalExceptionHandler.java
La clase con @ControllerAdvice que intercepta TODAS las excepciones y las convierte a JSON bonito.
// Lo que el cliente recibe (JSON):
{
  "timestamp": "2026-04-07T22:00:00",
  "message": "No tiene saldo disponible
   para vincularse al fondo FDO-ACCIONES"
}

// En vez de un stacktrace feo de 50 líneas 😱

💡 ¿Qué es Mockito?
Mockito crea "dobles" de los repositorios. En vez de ir a MongoDB de verdad, le decimos: "cuando alguien llame a findByUsername("test"), devuelve este usuario falso". Así probamos la lógica sin base de datos.
